This module aims to equip students with the skills and knowledge in order to apply state-of-the-art data science and software engineering techniques in a manufacturing context. Students will be introduced to the key statistical concepts that underpin a data-centric approach, such as uncertainty, variability, probability, hypothesis testing, and machine learning. Alongside this, the software engineering tools that are needed in order to apply these techniques are presented, for example: software and data management, version control, code design and configuration, testing, and reproducible software. The module also supports students to write a high quality research data management plan for their broader programme of research.

This module will be assessed via a written report, a coding portfolio and a research data management (RDM) plan produced in response to a manufacturing data problem. The data will be provided.

The report and coding portfolio will provide 80% of the total mark, while the RDM plan will provide 20% of the mark.

Feedback for the RDM will be given to provide support for the student's own research project.

By the end of this module, students will be able to:

**LO1:** Use relevant software tools to perform statistical analysis of manufacturing data, writing valid, understandable and reliable software/code that enables repeatable research.

**LO2:** Critically evaluate the use and relevance of different statistical methods and apply them in a manufacturing context.

**LO3:** Understand and use a Research Data Management (RDM) plan in the context of their own programme of research.

**Data Science Requirements (40% of the Module grade)**

- Use the information given to lay out the business case for the project and identify the stakeholders. 
- Identify sources of variability and uncertainty that may occur during the experimental period. How can you avoid it or work with it?
- Identify the experimental design and conditions – is the design sensible, are there limitations?  
- Produce a data pipeline from source to analysis - how do you handle missing data? 
- Use suitable analysis to identify the defects present in the dataset - which defects are most easily identified? 
- Use this example to make recommendations on data requirements and analysis methods to **identify defects in production**. Where could data collection be improved?  
- **Write a project report evidencing all previous points.**

**Research Software Requirements (40% of the Module grade)**

Using the data set proposed in the data science part, create a code in Python that satisfies the following requirements:

- Version control: Create a public [GitHub](http://github.com) repository to hold the code for your project (do **NOT** store the data on GitHub, just the code) and use version control to track the development of the code. 
- Design and interface: Your code should be based on a logical structure, using functions and classes for common tasks, and should use consistent style and naming conventions. Your code should contain a command line interface so it is possible to load different datasets in the software.
- Documentation and license: Provide both in-code documentation and a README that explains clearly the purpose of your software, how to install it, where to find the data and usage examples. Apply an appropriate license to your software and make sure that it is clearly highlighted in your README.
- Testing: Ensure that your software is validated using either unit testing or a test case based on a known dataset.
